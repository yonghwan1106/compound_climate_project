3.3 복합 극한기후 유형 정의 (AI 도출)

유형 구성 요소 탐지 기준 발생 메커니즘
Type A 폭염 + 가뭄 일최고기온 ≥33°C 연속 2일 & SPI < -1.5 동시 발생 (Multivariate)
Type B 폭염 + 열대야 일최고기온 ≥33°C & 일최저기온 ≥25°C 동시 발생 (Multivariate)
Type C 한파 + 대설 일최저기온 ≤-12°C & 일적설 ≥5cm 동시 발생 (Multivariate)
Type D 폭우 → 폭염 일강수량 ≥80mm 후 7일 이내 폭염 발생 순차 발생 (Temporally Compounding)
Type E 가뭄 → 폭우 SPI < -1.5 지속 후 일강수량 ≥80mm 순차 발생 (Temporally Compounding)

3.4 취약성 평가 프레임워크 (AI 설계)

Vulnerability Index = (Exposure × Sensitivity) / Adaptive Capacity • Exposure (노출도): 복합 극한기후 발생 빈도, 강도, 지속기간 • Sensitivity (민감도): 인구밀도, 고령인구비율, 농경지비율, 의료취약계층 • Adaptive Capacity (적응역량): 의료시설 수, 재해대응인력, 재정자립도

4.3 AI 모델 아키텍처 (AI 설계)

Model 1: Compound Event Transformer (탐지 모델)

입력: 다변량 기상 시계열 [기온, 강수, 습도, 기압, 풍속] × 30일 + 계절성 인코딩 (sin/cos positional encoding) + 지역 임베딩 아키텍처: ├── Input Embedding Layer (d_model=128) ├── Seasonal Positional Encoding ├── Multi-Head Self-Attention (8 heads) × 4 layers ├── Feed-Forward Network (hidden=512) ├── Layer Normalization + Dropout(0.1) └── Classification Head (5 classes) 출력: 복합 이벤트 유형 분류 (Type A~E + Normal) 파라미터 수: 807,559개

Model 2: Hybrid Impact Predictor (영향 예측 모델)

입력: 기후 변수 + 사회경제 변수 + 지역 특성 아키텍처: ├── XGBoost: Tabular 데이터의 비선형 관계 학습 ├── Neural Network: 복잡한 상호작용 포착 └── Ensemble: Weighted Average (XGB 0.6 + NN 0.4) 출력: 재산피해액, 건강영향(온열질환), 농업피해 + MC Dropout 불확실성 추정

4.4 분석 파이프라인 (AI 설계)

Phase 1: 데이터 수집 → Phase 2: 전처리 → Phase 3: 복합이벤트 탐지 ↓ ↓ ↓ 기상청 API 결측치 처리 Transformer 모델 ERA5 재분석 표준화/정규화 Rule-based 검증 KOSIS 통계 시계열 분할 Copula 확률 계산 ↓ ↓ ↓ Phase 4: 영향 예측 → Phase 5: 취약성 분석 → Phase 6: 시각화 ↓ ↓ ↓ XGBoost+NN IPCC 프레임워크 지도 시각화 Multi-task 시군구별 지수 트렌드 분석

5.3 AI 생성 코드 예시

# src/data_collection/kma_collector.py (Claude AI 작성) class KMADataCollector: """기상청 기상자료개방포털 데이터 수집기""" def __init__(self, api_key: str): self.api_key = api_key self.base_url = "https://apihub.kma.go.kr/api/typ01/url/kma_sfctm2.php" def collect_daily_weather(self, start_date, end_date, station_ids): """일별 기상관측 데이터 수집""" all_data = [] for stn_id in station_ids: params = { 'tm1': start_date.strftime('%Y%m%d'), 'tm2': end_date.strftime('%Y%m%d'), 'stn': stn_id, 'authKey': self.api_key } response = requests.get(self.base_url, params=params) data = self._parse_response(response.text) all_data.extend(data) return pd.DataFrame(all_data) def _handle_missing_values(self, df): """결측치 처리: 선형 보간 + 기후값 대체""" df = df.interpolate(method='linear', limit=3) df = df.fillna(df.groupby(['month']).transform('mean')) return df

6.3 AI 생성 핵심 코드

# src/models/transformer_detector.py (Claude AI 작성) class CompoundEventTransformer(nn.Module): """복합 극한기후 이벤트 탐지를 위한 Transformer 모델""" def __init__(self, config: TransformerConfig): super().__init__() self.input_projection = nn.Linear(config.input_dim, config.d_model) self.seasonal_encoding = SeasonalPositionalEncoding(config.d_model) encoder_layer = nn.TransformerEncoderLayer( d_model=config.d_model, nhead=config.n_heads, dim_feedforward=config.d_ff, dropout=config.dropout ) self.transformer = nn.TransformerEncoder(encoder_layer, config.n_layers) self.classifier = nn.Linear(config.d_model, config.num_classes) def forward(self, x, dates): x = self.input_projection(x) x = self.seasonal_encoding(x, dates) x = self.transformer(x) return self.classifier(x[:, -1, :]) # 마지막 시점 분류
